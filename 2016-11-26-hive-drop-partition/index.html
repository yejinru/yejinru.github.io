Hive Drop Partition

Posted on 2016-11-26

1. 背景

前几天生产任务突然慢了很多，查看下hdfs，发现从某天开始数据量比前一天几乎增加了一倍（这是要搞一个大新闻(╯‵□′)╯︵┴─┴）。

查数据时发现，某张hive表几乎有一半数据的key是重复的。

查看这张表的hdfs存储小文件，发现有从某个小文件开始，时间戳跟前面小文件的时间戳差一天（黑人问号脸）。因此怀疑就是这部分小文件搞的鬼。

查看下创建hive表的sql语句，创建的是外部表。

由于生产数据不能乱删，只能自己写sql来尝试和验证。

2. 重现步骤

  1. 本地创建一个test_file，文件内容：

    old

  2. 在hdfs上创建两个小文件，并往两文件中写些数据。

    dfs -put ${test_file} ${partition_dir}/000000_0;
dfs -put ${test_file} ${partition_dir}/000001_0;

dfs -ls ${hdfs_location}/${table_name}/log_date=${date};
    输出：
        -rw-r--r--   3 yjr yjr          4 2016-11-26 16:38 /user/yjr/yjr/yjr_test/log_date=2016-11-26/000000_0
        -rw-r--r--   3 yjr yjr          4 2016-11-26 16:38 /user/yjr/yjr/yjr_test/log_date=2016-11-26/000001_0

dfs -cat ${hdfs_location}/${table_name}/log_date=${date}/*;
    输出：
        old
        old

  3. 创建外部表。

    drop table if exists ${table_name};

create external table if not exists ${table_name} (
    user_id string
) partitioned by (log_date string)
ROW FORMAT DELIMITED FIELDS TERMINATED BY '\t'
LOCATION  '${hdfs_location}/${table_name}';

  4. 删除分区，insert overwrite。

    -- 1. 删除分区
alter table ${table_name} drop if exists partition(log_date='${date}');

-- 2. 插入数据
insert overwrite table ${table_name} partition (log_date='${date}')
select 'error';

-- 3. 查看hdfs文件
dfs -ls ${hdfs_location}/${table_name}/log_date=${date};
    输出：
        -rwxr-xr-x   3 yjr yjr          6 2016-11-26 16:38 /user/yjr/yjr/yjr_test/log_date=2016-11-26/000000_0
        -rw-r--r--   3 yjr yjr          4 2016-11-26 16:38 /user/yjr/yjr/yjr_test/log_date=2016-11-26/000001_0

dfs -cat ${hdfs_location}/${table_name}/log_date=${date}/*;
    输出：
        error
        old

-- 4. select行数
select * from ${table_name} where log_date='${date}';
    输出：
        error   2016-11-26
        old     2016-11-26

  5. 不删除分区，insert overwrite。

    -- 1. 插入数据
insert overwrite table ${table_name} partition (log_date='${date}')
select 'right';

-- 2. 查看hdfs文件
dfs -ls ${hdfs_location}/${table_name}/log_date=${date};
    输出：
        -rwxr-xr-x   3 yjr yjr          6 2016-11-26 16:38 /user/yjr/yjr/yjr_test/log_date=2016-11-26/000000_0

dfs -cat ${hdfs_location}/${table_name}/log_date=${date}/*;
    输出：
        right

-- 3. select行数
select * from ${table_name} where log_date='${date}';
    输出：
        right   2016-11-26

  6. 完整步骤：

    ----------------------------- 选择数据库 -----------------------------
use yjr;

----------------------------- 设置变量 -------------------------------
set hivevar:hdfs_location=/user/yjr/yjr;
set hivevar:table_name=yjr_test;
set hivevar:test_file=test_file;
set hivevar:date=2016-11-26;
set hivevar:table_dir=${hdfs_location}/${table_name};
set hivevar:partition_dir=${table_dir}/log_date=${date};

----------------------------- 插入数据 -------------------------------
dfs -put ${test_file} ${partition_dir}/000000_0;
dfs -put ${test_file} ${partition_dir}/000001_0;
dfs -ls ${partition_dir};
dfs -cat ${partition_dir}/*;

------------------------------- 删除表 --------------------------------
drop table if exists ${table_name};

create external table if not exists ${table_name} (
    user_id string
) partitioned by (log_date string)
ROW FORMAT DELIMITED FIELDS TERMINATED BY '\t'
LOCATION  '${table_dir}';

---------------------- 删除分区，insertoverwrite -----------------------
-- 1. 删除分区
alter table ${table_name} drop if exists partition(log_date='${date}');

-- 2. 插入数据
insert overwrite table ${table_name} partition (log_date='${date}')
select 'error';

-- 3. 查看hdfs文件
dfs -ls ${partition_dir};
dfs -cat ${partition_dir}/*;

-- 4. select行数
select * from ${table_name} where log_date='${date}';


---------------------- 不删除分区，insertoverwrite ---------------------
-- 1. 插入数据
insert overwrite table ${table_name} partition (log_date='${date}')
select 'right';

-- 2. 查看hdfs文件
dfs -ls ${partition_dir};
dfs -cat ${partition_dir}/*;

-- 3. select行数
select * from ${table_name} where log_date='${date}';

3. 原理

hive中分内部表（table, internal table）和外部表（external table）。

  1. 内部表：表数据和meta数据均存放在数据库中。当我们drop partition时，会将表数据和meta数据均从hdfs上删除。

  2. 外部表：meta数据存放在数据库中，但是表数据却存放在数据库之外别的位置。所以当我们drop partition时，只会把meta数据从hdfs上删除，表数据依然保留在hdfs上，只是meta数据删除后索引不到了而已，是逻辑上的删除而不是物理意义上的删除。

所以，当我们在insert overwrite前先drop partition，可能会出现问题。

4. 结论

使用hive外部表时：

  1. 如果想删除分区，千万不要使用drop partition。

  2. 在insert overwrite前也千万不要drop partition。
