Hive Drop Partition

Veröffentlicht am 2016-11-26

1. 背景

前几天生产任务突然慢了很多，查看下hdfs，发现从某天开始数据量比前一天几乎增加了一倍（这是要搞一个大新闻(╯‵□′)╯︵┴─┴）。

查数据时发现，某张hive表几乎有一半数据的key是重复的。

查看这张表的hdfs存储小文件，发现有从某个小文件开始，时间戳跟前面小文件的时间戳差一天（黑人问号脸）。因此怀疑就是这部分小文件搞的鬼。

查看下创建hive表的sql语句，创建的是外部表。

由于生产数据不能乱删，只能自己写sql来尝试和验证。

3. 原理

hive中分内部表（table, internal table）和外部表（external table）。

  1. 内部表：表数据和meta数据均存放在数据库中。当我们drop partition时，会将表数据和meta数据均从hdfs上删除。

  2. 外部表：meta数据存放在数据库中，但是表数据却存放在数据库之外别的位置。所以当我们drop partition时，只会把meta数据从hdfs上删除，表数据依然保留在hdfs上，只是meta数据删除后索引不到了而已，是逻辑上的删除而不是物理意义上的删除。

所以，当我们在insert overwrite前先drop partition，可能会出现问题。

4. 结论

使用hive外部表时：

  1. 如果想删除分区，千万不要使用drop partition。

  2. 在insert overwrite前也千万不要drop partition。

1. 第一课

  1. 监督学习：输入数据有标准答案。

    1. 回归问题：定量输出，连续变量预测。

      例子：房子大小和房子价格的关系。

    2. 分类问题：定性输出，离散变量输出。

      例子：癌症细胞大小决定是否患了癌症。

  2. 无监督学习：输入数据没有标准答案。

    例子：鸡尾酒宴会中，区分人声和背景声音。

  3. 强化学习：奖励或惩罚学习。

    例子：当狗做对了给狗奖励，当狗做错了给狗惩罚，最终狗会越来越听话。

  4. Notations:

    $$ x^{(i)} $$ : input variables，输入变量

    $$ y^{(i)} $$ : output(target) variables，输入变量

    $$ (x^{(i)}, y^{(i)}) $$ : training example，训练样本

    a list of m training examples { $$ (x^{(i)}, y^{(i)}) $$ ; i = 1, . . . ,m} is called a training set，训练集

    回归：y的取值是连续的

    分类：y的取值是一个很小的集合

    h: hypothesis, 假设函数

Webp Decode

Veröffentlicht am 2016-09-24

最近做了下webp解码成png、awebp（webp动画）解码为gif的功能，但是网上并没找到很合适的代码。

对于webp解码成png，应该比较好做，获取rgb，然后用gm对rgb数组进行解码即可。

对于awebp解码成gif，则需要一帧帧取出，然后再用gm把一帧帧存到gif中。

在libwebp库的demux.h头文件中，有说明如何取出awebp中的帧，具体代码如下：

WebPDemuxer* demux = WebPDemux(&webp_data);
uint32_t width = WebPDemuxGetI(demux, WEBP_FF_CANVAS_WIDTH);
uint32_t height = WebPDemuxGetI(demux, WEBP_FF_CANVAS_HEIGHT);
// ... (Get information about the features present in the WebP file).
uint32_t flags = WebPDemuxGetI(demux, WEBP_FF_FORMAT_FLAGS);
// ... (Iterate over all frames).
WebPIterator iter;
if (WebPDemuxGetFrame(demux, 1, &iter)) {
  do {
    // ... (Consume 'iter'; e.g. Decode 'iter.fragment' with WebPDecode(),
    // ... and get other frame properties like width, height, offsets etc.
    // ... see 'struct WebPIterator' below for more info).
  } while (WebPDemuxNextFrame(&iter));
  WebPDemuxReleaseIterator(&iter);
}
// ... (Extract metadata).
WebPChunkIterator chunk_iter;
if (flags & ICCP_FLAG) WebPDemuxGetChunk(demux, "ICCP", 1, &chunk_iter);
// ... (Consume the ICC profile in 'chunk_iter.chunk').
WebPDemuxReleaseChunkIterator(&chunk_iter);
if (flags & EXIF_FLAG) WebPDemuxGetChunk(demux, "EXIF", 1, &chunk_iter);
// ... (Consume the EXIF metadata in 'chunk_iter.chunk').
WebPDemuxReleaseChunkIterator(&chunk_iter);
if (flags & XMP_FLAG) WebPDemuxGetChunk(demux, "XMP ", 1, &chunk_iter);
// ... (Consume the XMP metadata in 'chunk_iter.chunk').
WebPDemuxReleaseChunkIterator(&chunk_iter);
WebPDemuxDelete(demux);

最后，我们用gm来对webp/awebp进行解码：

#include <webp/decode.h>
#include <webp/demux.h>
#include <Magick++.h>
#include <string>
#include <sstream>
#include <iostream>
#include <fstream>
#include <list>

class Webp {

public:
  static bool Decode(const std::string &content, const std::string &out_file) {
    WebPData webp_data;
    webp_data.bytes = reinterpret_cast<const uint8_t*>(content.c_str());
    webp_data.size = content.size();

    WebPDemuxer* demux = WebPDemux(&webp_data);
    int width = WebPDemuxGetI(demux, WEBP_FF_CANVAS_WIDTH);
    int height = WebPDemuxGetI(demux, WEBP_FF_CANVAS_HEIGHT);
    uint32_t flags = WebPDemuxGetI(demux, WEBP_FF_FORMAT_FLAGS);

    WebPIterator iter;
    int fragment_cnt = 0;
    std::list<Magick::Image> images;

    if (WebPDemuxGetFrame(demux, 1, &iter)) {
      do {
        ++ fragment_cnt;
        WebPData &fragment = iter.fragment;
        uint8_t *out_data = NULL;
        int length = 1;

        // 读取rgb/rgba
        if (iter.has_alpha) {
          out_data = WebPDecodeRGBA(fragment.bytes, fragment.size, &width, &height);
          length = width * height * 4;
        } else {
          out_data = WebPDecodeRGB(fragment.bytes, fragment.size, &width, &height);
          length = width * height * 3;
        }

        // 帧与帧之间必须设置延时，并且必须是帧与帧之间设置，否则设置会无效，导致转换后的gif变化很慢
        if (fragment_cnt > 1) {
          images.back().animationDelay(iter.duration / 10);
        }

        std::stringstream oss2;
        oss2 << width << "x" << height;

        images.push_back(Magick::Image());
        Magick::Image &image = images.back();
        image.size(oss2.str());
        image.magick(iter.has_alpha ? "RGBA" : "RGB");
        Magick::Blob ori_blob(out_data, length);
        image.read(ori_blob);
        WebPFree(out_data);
      } while (WebPDemuxNextFrame(&iter));
    }
    if (fragment_cnt == 1) {  // webp
      images.back().write(out_file.c_str());
    } else {  // awebp
      Magick::writeImages(images.begin(), images.end(), out_file.c_str());
    }
    WebPDemuxDelete(demux);
    return true;
  }
};

int main(int argc, char **argv) {
  Magick::InitializeMagick(NULL); // 对 gm 进行初始化
  std::ifstream t(argv[1]); // 读取文件
  std::string img_data((std::istreambuf_iterator<char>(t)), std::istreambuf_iterator<char>());
  Webp::Decode(img_data, argv[2]);
  return 0;
}
